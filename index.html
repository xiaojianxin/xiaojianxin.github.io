mysql心得(一)

发表于 2015-12-14

本周研究了一些数据库连接方面的知识

外连接

左外连接

1.语法

select a.,b. from a left join b on a.id = b.id;

2.效果

左外连接的结果集包括left join 指定的左表的所有行，而不仅仅是连接所匹配的行

右外连接

1.语法

select a.,b. from a right join b on a.id = b.id;

2.效果

右外连接的结果集包括right join 指定的右表的所有行，而不仅仅是连接所匹配的行

交叉连接

1.语法

select a.,b. from a cross join b;

2.效果

a表和b表的笛卡尔积

完全连接

1.语法

select a.,b. from a full join b ;

2.效果

两个表的全部数据都有，不匹配用null表示

内连接

典型的连接运算
1.语法

select a.,b. from a join b on a.id = b.id;

2.效果

利用内连接可获取两表的公共部分的记录。

nodejs 异步的两种方法（当然还有很多种，本次开发就用到两种）

1.async.mapSeries
用法async.mapSeries(arr, iterator, callback)

arr是一个数组，不能是一个json对象。

作用的将arr中的每一项依次拿给iterator去执行，执行结果传给最后的callback

  1. async.waterfall

用法：async.waterfall(tasks, [callback]);

task是函数组成的数组，callback是中途出错或者全部执行完后的回调函数。

它的特点是串行执行函数，并且前一个函数的结果会传给下一个函数，比较类似Step模块的功能。
