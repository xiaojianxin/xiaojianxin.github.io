新时代的php的新特性

命名空间

在学习yii2.0框架的时候经常看到这样的代码，在每个文件的头部，都会有像这样的一句话：

这就是命名空间，其作用就是按照一种虚拟的层次结构组织php代码，这种层次结构类似操作系统中文件系统的目录结构。

使用命名空间是现代php组件生态系统的基础，组件和框架作者编写了大量的代码，如果没有命名空间，开发者在使用的时候会发生名称的冲突，导致php执行出错。

声明命名空间：

同一命名空间或子命名空间没有必要在同一个php文件中声明，我们可以在不同文件中编写属于同一个命名空间的多个类，just like what YII2.0 does

导入函数和常量

全局命名空间

如果引用类，接口，函数或者常量时没有使用命名空间，php假定引用的类，接口，函数或者常量在当前命名空间中。

如果这个假定不正确，php就会尝试解析类，接口，函数或者常量。如果需要在命名空间中引用其他命名空间的类，接口，函数或者常量，必须使用完全限定的php类名（命名空间+类名）。

有些代码可能没有命名空间，这些代码在全局命名空间中，php原生的Exception类就是如此。

使用接口

接口是两个PHP对象之间的契约，其目的不是让一个对象依赖另一个对象的身份，而是依赖另一个对象的能力。接口把代码和依赖解耦了，而且允许我们的代码依赖任何实现了预期接口的第三方代码。

实例：

新建一个DocumentStore的PHP类，这个类的作用是从不同的源收集文本

这里的Documentable就是一个接口，而不是类

定义Documentable接口

如果要使用curl从远程URL获取HTML，则定义HtmlDocument类

如果要实现读取流资源，则定义StreamDocument类

如果要实现获取终端命令的执行结果的类


class CommandOutputDocument implements Documentable
{ 
    protected $command;

    public function _construct($command)
    {
        $this->command = $command;
    }

    public function getId()
    {
        return $this->command;
    }

    public function getContent()
    {
        return shell_exec($this->command);
    }
}

使用DocumentStore类


$documentStore = new DocumentStore();

//添加html文档；
$htmlDoc = new HtmlDocument('https://php.net');
$documentStore->addDocument($htmlDoc);

//添加流文档；
$streamDoc = new StreamDocument(fopen('stream.txt','rb'));
$documentStore->addDocument($streamDoc);

//添加终端命令文档
$cmdDoc = new CommandOutputDocument('cat /etc/hosts');
$documentStore->addDocument($cmdDoc);

print_r($documentStore->getDocument());
